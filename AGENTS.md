# AI Rules and Capabilities

You are helping build a professional developer portfolio built with Next.js 14.

## PROJECT TYPE
Developer Portfolio built with Next.js 14.

## TECH STACK
- Next.js 14 (App Router)
- TypeScript
- TailwindCSS
- Framer Motion
- React Icons

## RULES

1. Always follow a modular architecture.
2. Use Next.js App Router with TypeScript.
3. Use TailwindCSS for all styling.
4. Maintain a clean and predictable folder structure.
5. Follow this strict folder structure:
   - `/app`
   - `/components`
   - `/sections`
   - `/data`
   - `/types`
   - `/lib`
6. Separate UI into two layers:
   - UI components (buttons, cards, reusable elements)
   - Section components (Hero, About, Skills, Projects, etc.)
7. Keep components small, reusable, and focused on a single responsibility.
8. Do not hardcode portfolio data inside UI components.
9. All content must be stored in typed data files inside `/data` and consumed via props.
10. Ensure the architecture allows new sections and projects to be added easily.
11. Maintain a consistent dark theme design system.
12. Follow mobile-first responsive design.
13. Use Framer Motion only for subtle UI transitions such as:
    - fade
    - slide
    - hover
    - small section reveals
14. Avoid heavy or distracting animations.
15. Use consistent naming conventions:
    - Components → PascalCase
    - Files → kebab-case
    - Variables → camelCase
16. Optimize images using `next/image`.
17. Avoid unnecessary re-renders and keep components performant.
18. Ensure accessibility basics:
    - semantic HTML
    - alt attributes for images
    - keyboard navigation where appropriate
19. Prefer reusable UI components instead of duplicated layout code.
20. Ensure code remains clean, readable, and production-ready.

- Create responsive layouts using TailwindCSS
